# SBPO EXPERIMENTAL NOTEBOOKS

This directory contains the experimental notebooks and generated artifacts used to study XOPT on OR-Library p-median instances. The notebooks are organized as a connected experimental sequence: first validating the metaheuristic and exact baselines, then extracting structural interpretability signals, and finally testing whether those signals can support reoptimization under perturbed or disrupted scenarios.

The material in this folder should be read as experimental content for an article that combines these results into a single empirical narrative. Each notebook contributes one part of the evidence: solution quality, exact comparison, interpretability, and reoptimization behavior.

## Directory Contents

- `1. OR-Library Benchmark.ipynb`: Runs the `pymedian` metaheuristic on OR-Library p-median benchmark instances. It records one row per stochastic execution, aggregates metrics by instance, computes gaps against published best-known costs, and prepares publication-oriented LaTeX tables. This notebook establishes the metaheuristic performance baseline.

- `2. MIP Exact Benchmark.ipynb`: Solves the OR-Library p-median instances with an exact `python-mip` formulation. It reports solver status, objective value, runtime, selected facilities, and gaps to the published best-known values. This notebook provides the exact optimization baseline used to validate and contextualize the heuristic results.

- `3. Structural Interpretability Benchmark.ipynb`: Evaluates a post-hoc structural interpretability pipeline over the metaheuristic long-term memory. It builds a weighted co-occurrence graph from top long-term-memory solutions and extracts communities, max-p-cut partitions, highest k-core, and densest subgraph structures. It then computes metrics such as weighted modularity, coverage, best-facility concentration, best-facility separation, core recall, and densest-subgraph overlap. This notebook supplies the structural evidence used to explain patterns in high-quality solutions.

- `4. Optimizing Perturbed Instances.ipynb`: Tests reoptimization on perturbed p-median instances. It solves the original instance with the metaheuristic, extracts structural interpretability insights, perturbs graph edge costs, recomputes shortest-path distances, and reoptimizes the perturbed instance with exact MIP variants. The variants include unrestricted optimization and restrictions guided by max-p-cut, highest k-core, densest subgraph, and the union of highest k-core with densest subgraph. This notebook evaluates whether interpretability structures can guide efficient reoptimization after cost changes.

- `5. Exact Facility Removal Reoptimization.ipynb`: Studies a facility-removal scenario on the first OR-Library p-median instances. It solves the original instance exactly and heuristically, removes one facility from the exact optimal set, and reoptimizes while keeping the remaining `p - 1` facilities fixed. Replacement candidates are tested under baseline, highest k-core, densest subgraph, k-core/densest union, and max-p-cut strategies. This notebook evaluates whether structural insights can identify useful replacement pools after a disruption in the optimal solution.

- `lib/`: Local helper package shared by the notebooks. It includes utilities for locating project paths, reading OR-Library instances, building p-median MIP models, computing graph structures, running max-p-cut local search, extracting interpretability structures, and formatting metrics.

- `artifacts/`: CSV outputs generated by the notebooks. These files preserve raw executions, aggregated benchmark tables, structural interpretability metrics, perturbed-instance reoptimization results, and facility-removal reoptimization summaries.

## Generated Artifacts

- `pmedian_benchmark_raw.csv`: Execution-level metaheuristic benchmark results.
- `pmedian_benchmark_aggregated.csv`: Instance-level aggregated metaheuristic benchmark metrics.
- `mip_exact_results.csv`: Exact MIP results for OR-Library p-median instances.
- `structural_interpretability_benchmark.csv`: Structural metrics extracted from long-term-memory co-occurrence graphs.
- `reoptimization_interpretability_raw.csv`: Variant-level reoptimization results for perturbed instances.
- `reoptimization_interpretability_summary.csv`: Instance-level summary of perturbed-instance reoptimization.
- `reoptimization_interpretability_variant_aggregate.csv`: Cross-instance aggregate metrics by reoptimization variant.
- `facility_removal_reoptimization_raw.csv`: Variant-level results for the facility-removal experiment.
- `facility_removal_reoptimization_summary.csv`: Instance-level summary of the facility-removal experiment.
- `facility_removal_reoptimization_structures.csv`: Structural candidate sets and related data used in the removal experiment.
- `facility_removal_reoptimization_failures.csv`: Failure records, when present, for facility-removal variants.

## Article-Oriented Use

Together, these notebooks form a coherent experimental basis for an article about combining metaheuristic optimization, exact p-median models, and structural interpretability:

1. `1. OR-Library Benchmark.ipynb` shows how the metaheuristic performs across the benchmark set.
2. `2. MIP Exact Benchmark.ipynb` provides an exact reference for objective quality and runtime comparison.
3. `3. Structural Interpretability Benchmark.ipynb` transforms long-term-memory solution patterns into measurable graph structures.
4. `4. Optimizing Perturbed Instances.ipynb` tests those structures as candidate restrictions for reoptimization after cost perturbations.
5. `5. Exact Facility Removal Reoptimization.ipynb` tests the same structural ideas in a disruption scenario where an optimal facility is removed.

The intended article narrative is that high-quality solution memory is not only useful for benchmarking a metaheuristic, but can also be mined for interpretable structures that support constrained reoptimization. The artifacts in this folder provide the empirical tables needed to combine these claims into one experimental study.
